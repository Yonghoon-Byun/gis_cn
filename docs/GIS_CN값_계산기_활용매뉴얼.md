## 1장. 개요

### 1.1 플러그인 소개

GIS CN값 계산기는 QGIS 3.x 환경에서 동작하는 플러그인으로, 소유역 폴리곤을 입력받아 CN(Curve Number)값을 자동으로 산정한다. 수자원 설계 실무에서 유출량 산정의 핵심 매개변수인 CN값을 GIS 기반으로 체계적이고 반복 가능하게 계산하는 것이 목적이다.

### 1.2 주요 기능

- PostGIS 데이터베이스에서 수문학적 토양군(A/B/C/D)과 토지피복도를 자동 추출
- 소유역 폴리곤과의 GIS 교차 분석(Clip, Intersection)을 DB 측에서 수행하여 처리 효율 확보
- 토지피복도 원본 분류명을 CN값 테이블 항목에 맞게 재분류(매핑) 지원
- CN값 참조 테이블(cn_value.xlsx) 편집 및 커스터마이징
- AMC2(선행토양함수조건 II) 및 AMC3(선행토양함수조건 III) 자동 산정
- 소유역별 상세표(result1)와 요약표(result2)를 하나의 results.xlsx 파일로 내보내기

### 1.3 전체 워크플로우

플러그인은 4개의 탭으로 구성된 단계적 워크플로우를 따른다. 반드시 Tab 0부터 순서대로 실행해야 한다.

```
[소유역 폴리곤 입력]
       |
       v
[Tab 0: 레이어 불러오기]
  - PostGIS에서 토양군 추출 (Clip)
  - PostGIS에서 토지피복도 추출 (Clip)
  - PostGIS에서 토양군 x 토지피복도 교차 (Dissolve 포함)
  - QGIS에서 교차결과 x 소유역계 Intersection
       |
       v
[Tab 1: 토지이용 재분류]
  - 토지피복도 원본명 -> CN표 항목명 매핑
  - land_use_mapping.json 저장
       |
       v
[Tab 2: CN값 편집]
  - cn_value.xlsx 기반 CN값 테이블 확인/편집
  - 커스텀 CN표 불러오기/내보내기
       |
       v
[Tab 3: CN값 계산]
  - CN값_input 레이어 생성
  - 토지이용 재분류 매핑 적용
  - CN값 매칭 (토지이용분류 + 토양군 -> CN값)
  - results.xlsx 내보내기 (result1 + result2 시트)
```

## 2장. 설치 및 환경 설정

### 2.1 시스템 요구사항

| 항목 | 요구사항 |
|------|----------|
| QGIS 버전 | 3.0 이상 |
| 운영체제 | Windows |
| 네트워크 | 인터넷 연결 필수 (PostGIS 서버 접근) |
| 좌표계 | 입력 레이어 EPSG:5186 (Korea TM) |

### 2.2 외부 라이브러리

플러그인 실행 전 아래 Python 라이브러리가 QGIS Python 환경에 설치되어 있어야 한다.

| 라이브러리 | 용도 | 비고 |
|-----------|------|------|
| psycopg2 | PostGIS 데이터베이스 연결 | PostgreSQL 어댑터 |
| pandas | 데이터 처리 및 CN값 테이블 조작 | DataFrame 기반 |
| openpyxl | xlsx 파일 읽기/쓰기 | 결과 내보내기에 필수 |

설치 여부를 확인하려면 QGIS 메뉴에서 [플러그인] - [Python 콘솔]을 열고 다음을 입력한다.

```python
import psycopg2
import pandas
import openpyxl
print("모든 라이브러리 정상")
```

오류 없이 "모든 라이브러리 정상"이 출력되면 준비 완료이다. `ModuleNotFoundError`가 발생하는 경우, QGIS에 내장된 pip를 사용하여 해당 라이브러리를 설치한다.

### 2.3 설치 방법

#### 방법 1: 심볼릭 링크 (권장)

관리자 권한으로 명령 프롬프트(cmd)를 실행한 뒤 아래 명령어를 입력한다.

```
mklink /D "%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\gis_cn" "플러그인 실제 경로"
```

예시:

```
mklink /D "%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\gis_cn" "D:\DATA\gis_cn\gis_cn"
```

심볼릭 링크 생성 후 QGIS를 재시작하면 플러그인 목록에 "GIS CN값 계산기"가 나타난다.

#### 방법 2: 플러그인 경로 추가

1. QGIS 메뉴 - [플러그인] - [플러그인 관리 및 설치] 클릭
2. [설정] 탭 선택
3. [플러그인 경로] 항목에 플러그인이 위치한 **상위 폴더** 경로를 추가
4. [설치된 플러그인] 탭에서 "GIS CN값 계산기" 항목을 체크하여 활성화
5. QGIS 재시작

### 2.4 플러그인 실행

설치 완료 후 다음 방법으로 플러그인을 실행한다.

- **툴바**: 플러그인 아이콘 클릭
- **메뉴**: QGIS 메뉴에서 "CN값 계산기" 선택

실행하면 "CN값 계산기" 다이얼로그가 표시된다.

### 2.5 DB 연결 정보

플러그인은 아래 PostGIS 데이터베이스에 자동 연결된다. 별도의 DB 설정은 필요 없으며, 인터넷 연결만 확보하면 된다.

| 항목 | 값 |
|------|-----|
| Host | geo-spatial-hub.postgres.database.azure.com |
| Port | 6432 |
| Database | dde-water |
| Schema | public |
| 토양군 테이블 | public.soil (hydro_type: A/B/C/D) |
| 토지피복 테이블 | public.land_cover_yangju (l1/l2/l3 code/name) |
| 좌표계 | EPSG:5186 (Korea TM) |

플러그인 다이얼로그는 상단 헤더와 4개의 탭으로 구성된다.

```
+--------------------------------------------------+
| CN값 계산기 / 유역 CN값 산정 도구        [X 닫기] |
+--------------------------------------------------+
| [레이어 불러오기] [토지이용 재분류] [CN값 편집] [CN값 계산] |

|              (탭별 내용 영역)                       |

+--------------------------------------------------+
```

- **헤더**: 다이얼로그 상단, 흰색 배경, 하단 구분선

## 4장. 기능별 상세 사용법

### 4.1 레이어 불러오기 (Tab 0)

#### 입력 데이터 준비사항

- **소유역 폴리곤 파일**: SHP, GPKG 또는 GeoJSON 형식
- **좌표계**: 반드시 EPSG:5186 (Korea TM)이어야 한다. 다른 좌표계인 경우 QGIS 메뉴 [벡터] - [데이터 관리 도구] - [레이어 재투영]으로 미리 변환한다.
- **소유역명 속성**: 각 소유역을 구분할 수 있는 이름 필드가 포함되어야 한다.

#### 단계별 조작 방법

**1단계: 입력 방식 선택**

두 가지 방식 중 하나를 선택한다.

| 방식 | 조작 | 적용 상황 |
|------|------|-----------|
| 파일 선택 | [파일 선택] 라디오 버튼 클릭 - [찾아보기] 버튼으로 파일 탐색 | 프로젝트에 아직 로드하지 않은 파일 사용 시 |
| 현재 레이어 | [현재 레이어] 라디오 버튼 클릭 - 콤보박스에서 레이어 선택 | QGIS 캔버스에 이미 로드된 레이어 사용 시 |

**2단계: 소유역명 컬럼 선택**

레이어를 선택하면 "소유역명 컬럼" 콤보박스에 해당 레이어의 속성 필드 목록이 자동으로 표시된다. 소유역을 구분하는 필드를 선택한다. 이 값이 최종 결과 파일에서 소유역 이름으로 사용된다.

**3단계: 토지피복 분류 수준 선택**

| 분류 수준 | 라디오 버튼 | 설명 | 매핑 난이도 |
|-----------|------------|------|-------------|
| 대분류 | l1 | 시가화건조지역, 농업지역, 산림지역 등 | 쉬움 (항목 수 적음) |
| 중분류 | l2 | 주거지역, 상업지역, 논, 밭 등 | 보통 |
| 세분류 | l3 | 단독주거시설, 공동주거시설 등 | 어려움 (항목 수 많음) |

일반적으로 대분류(l1)가 매핑이 간편하다. 프로젝트 요구사항에 따라 선택한다.

**4단계: [실행] 클릭**

[실행] 버튼을 클릭하면 4단계의 백그라운드 처리가 자동으로 순서대로 진행된다.

| 단계 | 진행률 | 처리 내용 | 생성 레이어 | 비고 |
|------|--------|-----------|-------------|------|
| 1 | 3~22% | PostGIS에서 소유역 영역으로 토양군 Clip (ST_Intersection) | 토양군_clip | 캔버스에 즉시 추가 |
| 2 | 26~40% | PostGIS에서 소유역 영역으로 토지피복도 Clip (ST_Intersection) | 토지피복도_clip | 캔버스에 즉시 추가 |
| 3 | 44~68% | PostGIS에서 토양군 x 토지피복도 교차 (Dissolve 포함, 단일 CTE 쿼리) | 토양군_토지피복_교차 | 캔버스에 즉시 추가 |
| 4 | 72~95% | QGIS Processing으로 교차결과(3) x 소유역계 Intersection | (메모리 보관) | 캔버스에 추가하지 않음 |

각 단계가 완료될 때마다 해당 레이어가 QGIS 캔버스에 즉시 추가되어 시각적으로 확인할 수 있다.

**5단계: 완료 확인**

진행바가 100%가 되고 로그창에 "완료!" 메시지가 출력되면 처리가 끝난 것이다. 캔버스에 3개 레이어(토양군_clip, 토지피복도_clip, 토양군_토지피복_교차)가 추가되어 있는지 확인한다.

[다음 단계 ->] 버튼을 클릭하여 Tab 1(토지이용 재분류)로 이동한다.

#### 주의사항

- 소유역 레이어의 좌표계가 EPSG:5186이 아니면 결과가 부정확하다.
- PostGIS 서버에 접근 가능한 네트워크 환경인지 확인한다. 사내 방화벽이 포트 6432를 차단하는 경우 접속이 불가하다.
- 소유역 면적이 넓을수록 DB 쿼리 및 교차 연산에 시간이 소요된다. 로그창에서 진행 상황을 확인하며 기다린다.
- 처리 중 QGIS가 응답하지 않는 것처럼 보일 수 있으나 정상이다. 강제 종료하지 않는다.

### 4.2 토지이용 재분류 (Tab 1)

#### 목적

PostGIS에서 추출된 토지피복도의 원본 분류명(예: "자연림", "인공림")이 CN값 테이블(cn_value.xlsx)의 "토지이용분류" 항목명(예: "임야")과 다를 수 있다. 이 탭에서 원본명을 CN표 항목명에 맞게 매핑하여, CN값 매칭이 정상적으로 이루어지도록 한다.

#### 사용 방법

**1단계: [레이어에서 불러오기] 클릭**

버튼을 클릭하면 QGIS 캔버스에 있는 "토양군_토지피복_교차" 레이어에서 Tab 0에서 선택한 분류 수준(l1/l2/l3)에 해당하는 고유값 목록을 자동으로 추출하여 테이블의 "원본 이름" 열에 채운다.

기존에 입력했던 "재분류 이름"이 있는 경우 보존된다.

**2단계: 매핑 편집**

각 행의 "재분류 이름" 열에 cn_value.xlsx의 "토지이용분류" 항목명을 정확하게 입력한다.

매핑 예시 (대분류 기준):

| 원본 이름 | 재분류 이름 |
|-----------|-------------|
| 자연림 | 임야 |
| 인공림 | 임야 |
| 논 | 논 |
| 밭 | 밭 |
| 주거지역 | 주거지 |
| 상업지역 | 상업지 |
| 공업지역 | 공업지 |
| 초지 | 초지 |

편집 팁:
- 셀을 클릭하여 직접 입력한다.
- Enter 키를 누르면 오른쪽 다음 셀로 자동 이동한다. 행 끝에서 Enter를 누르면 다음 행 첫 셀로 이동한다.
- [+ 행 추가] 버튼으로 수동으로 항목을 추가할 수 있다.
- [- 행 삭제] 버튼으로 불필요한 항목을 삭제할 수 있다.

**3단계: [저장] 클릭**

매핑 내용이 플러그인 폴더 내 `land_use_mapping.json` 파일로 저장된다. 저장 완료 시 적용 항목 수가 표시된다.

저장된 매핑은 Tab 3의 CN값 계산 실행 시 자동으로 적용된다.

#### 핵심 규칙

- **재분류 이름은 cn_value.xlsx의 "토지이용분류" 컬럼과 정확히 일치해야 한다.** 대소문자, 띄어쓰기, 특수문자가 하나라도 다르면 CN값 매칭에 실패하여 해당 피처의 CN값이 NULL이 된다.
- 재분류 이름이 비어 있는 행은 저장에서 제외된다.
- [초기화] 버튼을 누르면 모든 매핑이 삭제되고 land_use_mapping.json도 비워진다. 확인 대화상자가 표시된다.

## 5장. 기술 참조

### 5.1 CN값 매칭 원리

CN값 테이블은 2차원 참조 테이블로, 행은 토지이용분류, 열은 수문학적 토양군(A/B/C/D)이다.

```
매칭 키: (토지이용분류명, 토양군코드) --> CN값(정수)
```

예시: ("임야", "B") --> 60

매칭 우선순위:

| 우선순위 | 데이터 소스 | 설명 |
|----------|------------|------|
| 1 (최우선) | Tab 2 위젯 테이블 | 다이얼로그에서 직접 편집한 CN값 |
| 2 (폴백) | cn_value.xlsx | Tab 2를 한 번도 열지 않았거나 테이블이 비어있는 경우 |

### 5.2 AMC2/AMC3 계산 공식

#### AMC2 (선행토양함수조건 II)

면적 가중평균으로 계산한다.

**토지이용별 AMC2:**

```
AMC2_lu = SUM(면적_토양군i * CN_토양군i) / SUM(면적_토양군i)
```

여기서 i는 토양군 A, B, C, D를 순회한다.

**소유역 전체 AMC2:**

```
AMC2_ws = SUM(전체_면적_피처j * CN_피처j) / 소유역_총면적
```

모든 토지이용과 토양군 조합의 면적-CN 곱의 합을 소유역 전체 면적으로 나눈다.

#### AMC3 (선행토양함수조건 III)

토지이용 유형에 따라 다르게 계산한다.

| 토지이용 | AMC3 계산식 | 비고 |
|---------|------------|------|
| 논, 답 | AMC3 = 79 (고정값) | 항상 79 적용 |
| 기타 모든 토지이용 | AMC3 = trunc(23 * AMC2 / (10 + 0.13 * AMC2)) | 정수 절사(trunc) |

- AMC2가 0 이하이거나 NaN인 경우 AMC3 = 0으로 처리한다.
- trunc는 소수점 이하를 버리는 절사 함수이다 (반올림 아님).

**소유역 전체 AMC3:**

```
AMC3_ws = SUM(AMC3_lu * 면적_lu) / 소유역_총면적
```

토지이용별 AMC3의 면적 가중평균이다. 결과는 소수점을 포함한 실수값이다.

### 5.3 PostGIS 공간 처리 구조

플러그인은 DB 측에서 공간 처리를 최대한 수행하여 QGIS Processing 단계를 최소화한다.

#### 토양군 Clip (단계 1)

```
ST_Intersects(인덱스 활용) --> ST_Intersection --> ST_CollectionExtract(3: Polygon만)
```

- 소유역 전체 Union 영역과 공간적으로 겹치는 토양군 피처를 필터링
- ST_Intersection으로 소유역 내부 영역만 잘라냄
- 결과가 MultiGeometry인 경우 Polygon 부분만 추출

#### 토지피복도 Clip (단계 2)

토양군 Clip과 동일한 구조를 사용한다.

#### 토양군 x 토지피복도 교차 (단계 3)

단일 CTE(Common Table Expression) 쿼리로 5단계를 일괄 처리한다.

| CTE 단계 | 이름 | 처리 내용 |
|----------|------|-----------|
| 1 | sc | 토양군 Clip (ST_Intersection) |
| 2 | sc_valid | 유효 geometry 필터 (NULL, Empty, Area=0 제거) |
| 3 | lc_raw | 토지피복도 Clip (ST_Intersection) |
| 4 | lc | 선택한 분류 수준 기준 ST_Union (Dissolve) |
| 5 | ix | sc_valid x lc ST_Intersection (최종 교차) |

이 방식은 QGIS의 `native:dissolve` + `native:intersection` 2단계를 대체하며, 네트워크 왕복 횟수를 줄여 처리 속도를 개선한다.

Dissolve 기준 컬럼은 분류 수준에 따라 결정된다:

| 분류 수준 | Dissolve 기준 컬럼 |
|-----------|-------------------|
| l1 | l1_code, l1_name |
| l2 | l2_code, l2_name |
| l3 | l3_code, l3_name |

#### QGIS Intersection (단계 4)

DB에서 생성된 토양군 x 토지피복도 교차 레이어와 원본 소유역 폴리곤을 QGIS Processing `native:intersection`으로 교차한다. 이 단계에서 각 피처에 소유역명 속성이 부여된다.

### 5.4 생성 레이어 목록

## 7장. 자주 묻는 질문 (FAQ)

**Q1. 토지피복도와 토양도 데이터는 어디서 가져오나요?**

PostGIS 데이터베이스(Azure)에 사전 적재되어 있다. 별도로 데이터를 준비하거나 다운로드할 필요가 없다. 사용자는 소유역 폴리곤만 준비하면 된다.

---

**Q2. 분류 수준(대/중/세분류)은 어떤 걸 선택해야 하나요?**

프로젝트 요구사항에 따라 결정한다.

| 분류 수준 | 장점 | 단점 |
|-----------|------|------|
| 대분류(l1) | 항목이 적어 매핑이 쉬움. 일반적인 수문 분석에 적합 | 세밀한 구분 불가 |
| 중분류(l2) | 적절한 상세도 | 매핑 항목이 중간 수준 |
| 세분류(l3) | 가장 정밀한 토지이용 구분 가능 | 매핑 항목이 많아 작업량 증가 |

특별한 사유가 없으면 대분류(l1)를 권장한다.

---

**Q3. CN값을 수정하고 싶은데 원본 파일이 바뀌나요?**

아니다. Tab 2에서 편집한 내용은 다이얼로그 내부의 테이블 위젯에만 적용된다. cn_value.xlsx 원본 파일은 변경되지 않는다. [내보내기] 버튼으로 별도 파일(cn_value_custom.xlsx)을 저장할 수 있으며, [기본값] 버튼으로 언제든 원본을 복원할 수 있다.

---

**Q4. 토지이용 재분류를 안 하면 어떻게 되나요?**

토지피복도의 원본 분류명이 cn_value.xlsx의 "토지이용분류"와 정확히 일치하면 재분류 없이도 CN값 매칭이 정상 작동한다. 그러나 원본명과 CN표 항목명이 다른 경우(대부분) 매칭에 실패하여 해당 피처의 CN값이 NULL이 된다.

---

**Q5. results.xlsx 파일 하나에 두 시트가 포함되나요?**

그렇다. result1(소유역별 토지이용 x 토양군 상세표)과 result2(유역별 CN값 요약표) 두 개 시트가 하나의 results.xlsx 파일에 저장된다.

---

**Q6. 소유역 레이어의 좌표계가 EPSG:5186이 아닌 경우?**

QGIS에서 미리 재투영하여 EPSG:5186으로 변환한 뒤 사용해야 한다. QGIS 메뉴에서 [벡터] - [데이터 관리 도구] - [레이어 재투영]을 사용하거나, Processing 도구상자에서 "Reproject layer"를 검색하여 실행한다.

---

**Q7. 여러 프로젝트에서 동일한 CN표를 쓰고 싶은데?**

Tab 2에서 [내보내기]로 xlsx 파일을 저장해 둔다. 다른 프로젝트에서 플러그인을 실행한 후 Tab 2에서 [불러오기]로 해당 파일을 로드하면 동일한 CN표를 재사용할 수 있다.

---

**Q8. 매핑(land_use_mapping.json)은 어디에 저장되나요?**

플러그인 폴더 루트에 저장된다. 플러그인을 재설치하거나 업데이트하면 이 파일이 삭제될 수 있으므로, 중요한 매핑은 별도 폴더에 백업해 둘 것을 권장한다.

---

**Q9. 다이얼로그를 닫았다가 다시 열면 이전 작업이 유지되나요?**

아니다. 다이얼로그를 닫으면 내부 상태(교차 레이어, CN 테이블 편집 내용 등)가 모두 초기화된다. 단, land_use_mapping.json에 저장된 매핑은 파일로 유지되므로 다음 실행 시 자동으로 로드된다.

---

**Q10. 동시에 여러 개의 소유역 파일을 처리할 수 있나요?**

현재 버전에서는 한 번에 하나의 소유역 폴리곤 파일만 처리할 수 있다. 여러 소유역을 처리하려면 하나의 파일에 모든 소유역 폴리곤을 포함시키되, 각 폴리곤에 고유한 소유역명 속성을 부여하면 된다.

## 8장. 부록

### 8.1 토지피복 분류 체계

토지피복도는 대분류(l1), 중분류(l2), 세분류(l3)의 3단계 계층 구조를 가진다. 각 수준에는 코드와 이름 쌍이 있다.

| 분류 수준 | 코드 컬럼 | 이름 컬럼 | 예시 코드 | 예시 이름 |
|-----------|-----------|-----------|-----------|-----------|
| 대분류(l1) | l1_code | l1_name | 100 | 시가화건조지역 |
| 중분류(l2) | l2_code | l2_name | 110 | 주거지역 |
| 세분류(l3) | l3_code | l3_name | 111 | 단독주거시설 |

대분류-중분류-세분류는 상위-하위 관계이다. 예를 들어, 대분류 "시가화건조지역(100)" 아래에 중분류 "주거지역(110)", "상업지역(120)" 등이 포함되고, 중분류 "주거지역(110)" 아래에 세분류 "단독주거시설(111)", "공동주거시설(112)" 등이 포함된다.

### 8.2 수문학적 토양군 분류

| 토양군 | 특성 | 투수성 | CN값 경향 |
|--------|------|--------|-----------|
| A | 모래질, 사력질 토양. 깊은 토층 | 높음 (침투율 높음) | 낮은 CN값 (유출 적음) |
| B | 사양토, 양토. 보통 깊이 토층 | 중간 | 중간 CN값 |
| C | 식양토, 미사질양토. 침투 억제층 존재 | 낮음 | 높은 CN값 |
| D | 점토, 불투수층 근접. 매우 얕은 토층 | 매우 낮음 (유출율 높음) | 가장 높은 CN값 (유출 많음) |

토양군이 A에서 D로 갈수록 투수성이 낮아지고 유출량이 증가한다. 동일한 토지이용이라도 토양군에 따라 CN값이 달라진다.

### 8.4 용어 정리

| 용어 | 설명 |
|------|------|
| CN (Curve Number) | 미국 SCS(토양보전청)에서 개발한 유출곡선번호. 0~100의 정수로 표현하며, 값이 클수록 유출량이 많다 |
| AMC (Antecedent Moisture Condition) | 선행토양함수조건. 강우 전 토양의 습윤 상태를 나타내는 지표 |
| AMC2 | 보통 조건의 선행토양함수조건. 면적 가중평균 CN값 |
| AMC3 | 습윤 조건의 선행토양함수조건. AMC2를 기반으로 보정한 CN값 |
| PostGIS | PostgreSQL 데이터베이스의 공간 확장. SQL로 GIS 공간 연산을 수행할 수 있다 |
| Clip | 한 레이어를 다른 레이어의 경계로 잘라내는 공간 연산 |
| Intersection | 두 레이어의 교차 영역을 추출하고 양쪽 속성을 결합하는 공간 연산 |
| Dissolve | 지정한 속성값이 동일한 인접 피처를 하나로 병합하는 공간 연산 |
| EPSG:5186 | 한국 TM(Transverse Mercator) 좌표계. 한반도 중부원점 기준, 단위: 미터 |
| 소유역 | 하천 유역을 수문학적으로 세분한 단위 유역 |
