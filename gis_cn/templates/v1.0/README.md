# HWP 템플릿 스펙 v1.0

이 디렉토리에는 플러그인이 출력하는 HWP 보고서의 **빈 양식 템플릿**(`cn_report.hwp`)이 들어갑니다.
첨부 샘플 `제4장 재해영향 예측 및 평가_최종.hwpx`의 **물리 페이지 25~34** 형식을 기반으로 합니다.

스키마 버전은 `gis_cn/core/analysis_result.py`의 `SCHEMA_VERSION` 상수와 **반드시 동기화**되어야 합니다.

---

## 1. 파일

| 파일 | 설명 |
|---|---|
| `cn_report.hwp` | 본체 템플릿 (한글에서 수동 제작 — 본 문서의 지시대로) |
| `README.md` | 본 스펙 문서 |

## 2. 페이지 구성 (레퍼런스 p.25~34 대응)

```
┌─ 페이지 1 ─────────────────────────────────────────┐
│ 표지 영역                                            │
│  누름틀: meta.project_name                           │
│  누름틀: meta.site_name                              │
│  누름틀: meta.author                                 │
│  누름틀: meta.organization                           │
│  누름틀: meta.analysis_date                          │
│  누름틀: meta.development_stage                      │
└─────────────────────────────────────────────────────┘

┌─ 페이지 2 (샘플 p.25 대응) ───────────────────────┐
│ "[표 4-N] 유출곡선지수(CN) 기준표 (AMC-II, Ia=0.2S)" │
│  표: tbl_cn_reference                                │
│    1행: 토지이용분류 | A | B | C | D  (고정 헤더)    │
│    2행 이후: 셀필드 lu | cn_a | cn_b | cn_c | cn_d   │
│ "[자료] 수치토지피복도 분류기준"                     │
└─────────────────────────────────────────────────────┘

┌─ 페이지 3~ (샘플 p.26~30 대응) ──────────────────┐
│ "[표 4-19] 개발단계별 유출곡선지수(CN) 산정결과"     │
│  표: tbl_cn_result                                   │
│    복합 헤더:                                         │
│      유역구분 | 토지이용분류                          │
│      | TYPE A (면적·CN) | TYPE B | TYPE C | TYPE D   │
│      | 총면적 | AMC조건 (Ⅱ CN / Ⅲ CN)                 │
│    데이터 행 1개(템플릿): 셀필드                      │
│      ws | lu                                          │
│      a_area | a_cn | b_area | b_cn                    │
│      c_area | c_cn | d_area | d_cn                    │
│      total_area | amc2_cn | amc3_cn                   │
│ "[자료]" / "[주]"                                     │
└─────────────────────────────────────────────────────┘

┌─ 페이지 요약 ─────────────────────────────────────┐
│ "[표 4-] 유역별 CN정리"                              │
│  표: tbl_cn_summary                                   │
│    헤더: 유역 | 총면적 | AMC2 CN | AMC3 CN            │
│    행 1개(템플릿): 셀필드 ws | area | amc2 | amc3     │
└─────────────────────────────────────────────────────┘

┌─ (선택) 유역합성 ─────────────────────────────────┐
│  표: tbl_cn_composite          (상세)                │
│  표: tbl_cn_composite_summary  (요약)                │
│  ※ 없으면 "해당 없음"으로 표시되도록 빈 템플릿만 두기 │
└─────────────────────────────────────────────────────┘

┌─ (선택) 지도/비고 ────────────────────────────────┐
│  책갈피: bm_map_0, bm_map_1, ...                     │
│    해당 위치에 InsertPicture로 지도 PNG 삽입          │
│  책갈피: bm_null_rows                                │
│    CN 매칭 실패 목록 텍스트 삽입                      │
│  누름틀: notes                                        │
└─────────────────────────────────────────────────────┘
```

## 3. 누름틀(메타) 이름 — 전부 전역 유니크

| 이름 | 내용 | 값 예 |
|---|---|---|
| `meta.project_name` | 과업명 | "OO 재해영향평가" |
| `meta.site_name` | 사업지구명 | "양주시 장흥지구" |
| `meta.author` | 작성자 | "홍길동" |
| `meta.organization` | 기관명 | "도화엔지니어링" |
| `meta.analysis_date` | 분석일자 (YYYY-MM-DD) | "2026-04-22" |
| `meta.development_stage` | 개발단계 | "현 상태" / "개발 전" / "개발 후" |
| `notes` | 비고/주석 | (다줄 텍스트) |

**한글 사용법**: 입력 → 필드 → 누름틀 삽입 → 이름 필드에 위 ID 기재.

## 4. 표(셀 필드) — 행은 **1행만** 두고 렌더러가 복제

### 4.1 `tbl_cn_reference` (p.25 기준표)

| 열 | 셀 필드 이름 | 값 |
|---|---|---|
| 1 | `lu`   | 토지이용분류명 |
| 2 | `cn_a` | A형 CN |
| 3 | `cn_b` | B형 CN |
| 4 | `cn_c` | C형 CN |
| 5 | `cn_d` | D형 CN |

### 4.2 `tbl_cn_result` (p.26~ 산정결과)

| 열 | 셀 필드 | 값 |
|---|---|---|
| 1 | `ws` | 소유역명 (또는 "합계") |
| 2 | `lu` | 토지이용 (합계행에는 "합계") |
| 3, 4 | `a_area`, `a_cn` | A형 면적·CN |
| 5, 6 | `b_area`, `b_cn` | B형 면적·CN |
| 7, 8 | `c_area`, `c_cn` | C형 면적·CN |
| 9, 10 | `d_area`, `d_cn` | D형 면적·CN |
| 11 | `total_area` | 총면적 |
| 12 | `amc2_cn` | AMC-II CN |
| 13 | `amc3_cn` | AMC-III CN |

### 4.3 `tbl_cn_summary` (유역별 CN정리)

| 열 | 셀 필드 | 값 |
|---|---|---|
| 1 | `ws`   | 유역명 |
| 2 | `area` | 총면적 |
| 3 | `amc2` | AMC2 CN |
| 4 | `amc3` | AMC3 CN |

### 4.4 `tbl_cn_composite` / `tbl_cn_composite_summary`

위 두 표와 **동일한 셀 필드 구조**. 유역합성 결과가 있을 때만 채워짐.

## 5. 책갈피 (이미지 삽입 위치)

| 이름 | 용도 |
|---|---|
| `bm_map_0`, `bm_map_1`, ... | 지도 이미지 N개를 순서대로 삽입 |
| `bm_null_rows` | CN 매칭 실패 목록 삽입 |

캡션용 누름틀은 `caption_0`, `caption_1`, ... 로 각 이미지 아래에 배치.

**한글 사용법**: 입력 → 책갈피 → 이름 기입.

---

## 6. 템플릿 제작 절차 (한글에서)

1. 새 HWP 문서 생성 → 용지 A4, 머리말/꼬리말 설정
2. 샘플 파일 p.25~34를 **빈 서식으로 모방** (숫자·결과값은 모두 비우기)
3. 표 삽입 후 각 셀 속성에서 **필드 이름** 설정 (표 4.1~4.4 스펙대로)
4. 메타 영역에 **누름틀** 삽입 (섹션 3 ID로)
5. 이미지 삽입 위치에 **책갈피** 추가
6. 표에 **데이터 행 1개만** 남기고 저장 (렌더러가 행 복제)
7. `cn_report.hwp` 이름으로 이 디렉토리에 저장

## 7. 검증

템플릿 제작 후 다음을 확인:

- [ ] 한글 파일 정상 열림 / 저장
- [ ] 플러그인에서 "HWP 내보내기" 실행 시 오류 없이 산출
- [ ] 표 행 수가 데이터 수에 맞게 자동 복제
- [ ] 합계 행이 각 소유역 블록 뒤에 삽입됨
- [ ] 유역합성 결과가 있을 때만 해당 표가 채워짐
- [ ] 지도 이미지가 올바른 책갈피 위치에 삽입됨
