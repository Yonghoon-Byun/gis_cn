# GIS CN값 계산기 사용법 가이드

**플러그인명**: GIS CN값 계산기
**버전**: 0.1 (experimental)
**저자**: 도화엔지니어링 기술개발연구원
**QGIS 최소버전**: 3.0
**좌표계**: EPSG:5186 (Korea TM)

---

## 개요

이 플러그인은 소유역 폴리곤을 기반으로 PostGIS 데이터베이스(토양군, 토지피복도)를 활용하여 CN값을 자동 계산하는 QGIS 플러그인입니다. 4개의 탭으로 구성된 단계적 워크플로우로 운영되며, 각 탭을 순서대로 실행해야 정상적인 결과를 얻을 수 있습니다.

---

## 설치 방법

### 방법 1: 심볼릭 링크 (권장)

관리자 권한으로 명령 프롬프트를 열고 아래 명령어를 실행합니다.

```
mklink /D "%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\gis_cn" "플러그인 실제 경로"
```

예시:
```
mklink /D "%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\gis_cn" "D:\DATA\gis_cn\gis_cn"
```

### 방법 2: 플러그인 경로 추가

1. QGIS 메뉴 → **플러그인** → **플러그인 관리 및 설치**
2. **설정** 탭 클릭
3. **플러그인 경로** 항목에 플러그인 상위 폴더 경로 추가
4. **설치된 플러그인** 탭에서 "GIS CN값 계산기" 항목 체크하여 활성화

### 외부 라이브러리 설치 확인

플러그인 실행 전 다음 Python 라이브러리가 설치되어 있어야 합니다.

| 라이브러리 | 용도 |
|-----------|------|
| psycopg2 | PostGIS 데이터베이스 연결 |
| pandas | 데이터 처리 |
| openpyxl | xlsx 파일 읽기/쓰기 |

QGIS Python 콘솔에서 설치 여부 확인:
```python
import psycopg2
import pandas
import openpyxl
```

## 워크플로우 개요

```
탭 0: 레이어 불러오기
  → 소유역 폴리곤 입력 및 PostGIS 데이터 클리핑

탭 1: 토지이용 재분류
  → 토지피복 원본값을 CN값 테이블 항목에 맞게 매핑

탭 2: CN값 편집
  → 토지이용분류별 토양군(A/B/C/D) CN값 확인 및 편집

탭 3: CN값 계산
  → 최종 CN값 계산 및 결과 xlsx 파일 내보내기
```

---

## 탭 0: 레이어 불러오기

### 목적

소유역 폴리곤을 입력받아 PostGIS 데이터베이스에서 토양군과 토지피복도를 클리핑하고, 교차 분석 레이어를 생성합니다.

### 사용 순서

**1단계: 소유역 레이어 입력**

- **파일에서 불러오기**: [파일 선택] 버튼 클릭 후 SHP, GPKG, GeoJSON 파일 선택
- **현재 QGIS 레이어 사용**: 콤보박스에서 QGIS 캔버스에 이미 로드된 레이어 선택

**2단계: 소유역명 컬럼 선택**

- 소유역을 구분하는 컬럼명을 콤보박스에서 선택합니다.
- 이 컬럼 값이 최종 결과 파일의 소유역 이름으로 사용됩니다.

**3단계: 토지피복 분류 수준 선택**

| 선택값 | 설명 |
|--------|------|
| l1 | 대분류 (예: 시가지, 농지, 산림) |
| l2 | 중분류 (예: 주거지역, 상업지역) |
| l3 | 세분류 (가장 세밀한 분류) |

> 분류 수준이 세밀할수록 탭 1에서 매핑해야 할 항목이 많아집니다.

**4단계: 실행**

[실행] 버튼 클릭 시 다음 4단계가 순서대로 자동 실행됩니다.

| 단계 | 작업 내용 | 생성 레이어 |
|------|-----------|-------------|
| 1 | PostGIS 토양군 Clip (ST_Intersection) | 토양군_clip |
| 2 | PostGIS 토지피복도 Clip | 토지피복도_clip |
| 3 | PostGIS 토양군 × 토지피복도 교차 (Dissolve 포함) | 토양군_토지피복_교차 |
| 4 | QGIS 교차 결과 × 소유역계 Intersection | (메모리 보관) |

- 진행 상황은 진행바와 하단 로그창에서 실시간으로 확인할 수 있습니다.
- 처리 중에는 QGIS가 응답하지 않는 것처럼 보일 수 있으나 정상입니다. 로그창을 확인하며 기다립니다.

**5단계: 다음 탭 이동**

완료 후 [다음 단계 →] 버튼이 활성화됩니다. 클릭하여 탭 1로 이동합니다.

### 주의사항

- 소유역 레이어의 좌표계는 **EPSG:5186** 이어야 합니다. 다른 좌표계인 경우 QGIS에서 미리 재투영하여 사용합니다.
- PostGIS 서버에 접근 가능한 네트워크 환경인지 확인합니다.
- 소유역 면적이 클수록 처리 시간이 길어집니다.

## 탭 1: 토지이용 재분류

### 목적

PostGIS 토지피복도의 원본 분류명을 cn_value.xlsx에 정의된 "토지이용분류" 항목에 맞게 재매핑합니다. 이 매핑이 없으면 CN값을 찾을 수 없습니다.

### 사용 순서

**1단계: 원본값 불러오기**

[레이어에서 불러오기] 버튼 클릭 시 "토양군_토지피복_교차" 레이어에서 탭 0에서 선택한 분류 수준(l1/l2/l3)의 고유값 목록을 자동으로 불러옵니다.

**2단계: 매핑 편집**

테이블에 두 개의 컬럼이 표시됩니다.

| 컬럼 | 설명 |
|------|------|
| 원본 이름 | 토지피복도에서 가져온 원래 분류명 (수정 불가) |
| 재분류 이름 | cn_value.xlsx의 토지이용분류 항목명 (직접 입력) |

- 재분류 이름은 cn_value.xlsx의 **"토지이용분류" 컬럼 값과 정확히 일치**해야 합니다. 오탈자에 주의합니다.
- 행 추가/삭제 버튼으로 항목을 편집할 수 있습니다.

**3단계: 저장**

[저장] 버튼 클릭 시 매핑 내용이 플러그인 폴더 내 `land_use_mapping.json` 파일로 저장됩니다. 저장된 매핑은 탭 3 CN값 계산 시 자동으로 적용됩니다.

### 버튼 설명

| 버튼 | 기능 |
|------|------|
| 레이어에서 불러오기 | 교차 레이어의 고유값을 테이블에 자동 입력 |
| 저장 | 현재 매핑을 land_use_mapping.json으로 저장 |
| 초기화 | 테이블의 모든 매핑 삭제 |

### 주의사항

- 재분류 이름이 비어 있는 항목은 CN값 계산 시 NULL 처리됩니다.
- 탭 0을 다시 실행하거나 분류 수준을 변경한 경우, 반드시 [레이어에서 불러오기]를 다시 실행하여 최신 고유값 목록을 갱신합니다.

---

## 탭 2: CN값 편집

### 목적

토지이용분류별 수문학적 토양군(A/B/C/D)에 따른 CN값 테이블을 확인하고 필요 시 편집합니다.

### 초기 로드

탭에 처음 진입하면 플러그인 폴더의 `cn_value.xlsx` 파일이 자동으로 로드됩니다.

### 테이블 구조

| 토지이용분류 | A | B | C | D |
|------------|---|---|---|---|
| 논/답 | ... | ... | ... | ... |
| 밭 | ... | ... | ... | ... |
| 산림 | ... | ... | ... | ... |
| ... | | | | |

- A/B/C/D는 수문학적 토양군을 의미합니다.
- 각 셀에는 해당 토지이용과 토양군 조합의 CN값(정수)이 입력됩니다.

### 편집 방법

- 셀을 더블클릭하면 직접 편집할 수 있습니다.
- Enter 키를 누르면 아래쪽 셀로 자동 이동합니다.

### 버튼 설명

| 버튼 | 기능 |
|------|------|
| + 행 추가 | 테이블 하단에 빈 행 추가 |
| - 행 삭제 | 선택한 행 삭제 |
| + 열 추가 | 테이블 오른쪽에 빈 열 추가 |
| 기본값 | cn_value.xlsx 원본 데이터로 초기화 |
| 불러오기 | 외부 .xlsx 파일에서 CN값 테이블 로드 |
| 내보내기 | 현재 편집 내용을 .xlsx 파일로 저장 |

### 주의사항

- CN값 계산 시 이 탭에서 편집 중인 테이블 데이터가 우선 사용됩니다.
- 토지이용분류 항목명은 탭 1의 재분류 이름과 정확히 일치해야 합니다.
- 편집 내용을 영구적으로 보존하려면 [내보내기]로 저장해 두세요.

## 탭 3: CN값 계산

### 목적

탭 0~2에서 준비된 데이터를 바탕으로 최종 CN값을 계산하고 결과를 xlsx 파일로 내보냅니다.

### 사용 순서

**1단계: CN값 계산 실행**

[CN값 계산 실행] 버튼 클릭 시 3단계가 순서대로 실행됩니다.

| 단계 | 작업 내용 |
|------|-----------|
| 5 | 교차 레이어 기반 CN값_input 레이어 구성 (소유역명, 토지이용, 유역면적, 토양군, cn값) |
| 6 | 토지이용 재분류 매핑 적용 (탭 1의 land_use_mapping.json 적용) |
| 7 | CN값 매칭 (토지이용분류 + 토양군 → 탭 2 테이블에서 CN값 조회) |

완료되면 "CN값_input" 레이어가 QGIS 캔버스에 추가됩니다.

> CN값이 NULL인 피처가 존재하는 경우 경고 팝업이 표시됩니다. 탭 1의 매핑과 탭 2의 CN값 테이블을 점검하십시오.

**2단계: 출력 설정**

- **CN값_input 레이어 선택**: 콤보박스에서 1단계에서 생성된 "CN값_input" 레이어를 선택합니다.
- **결과 저장 폴더 지정**: [폴더 선택] 버튼으로 결과 파일을 저장할 폴더를 지정합니다.

**3단계: 결과 내보내기**

[결과 내보내기] 버튼 클릭 시 지정한 폴더에 `results.xlsx` 파일이 생성됩니다.

### 결과 파일 구조 (results.xlsx)

**result1 시트: 상세 결과**

소유역별 토지이용 × 토양군(TYPE A/B/C/D) 조합별 상세 데이터와 AMC2/AMC3 CN값이 포함됩니다.

| 컬럼 | 설명 |
|------|------|
| 소유역명 | 탭 0에서 선택한 소유역명 컬럼 값 |
| 토지이용 | 재분류된 토지이용명 |
| TYPE A/B/C/D 면적 | 각 토양군별 면적 (m2) |
| CN (A/B/C/D) | 각 토양군별 CN값 |
| AMC2 CN | 면적 가중평균 CN값 |
| AMC3 CN | AMC3 보정 CN값 |

**result2 시트: 유역별 요약**

| 컬럼 | 설명 |
|------|------|
| 유역 | 소유역명 |
| 총면적 | 유역 전체 면적 (m2) |
| AMC2 CN | 유역 전체 AMC2 CN값 |
| AMC3 CN | 유역 전체 AMC3 CN값 |

---

## AMC2 / AMC3 계산 방법

### AMC2 (선행토양함수조건 II)

면적 가중평균으로 계산합니다.

```
AMC2 = sum(CN_i × Area_i) / sum(Area_i)
```

### AMC3 (선행토양함수조건 III)

토지이용 유형에 따라 다르게 계산됩니다.

| 토지이용 | AMC3 계산식 |
|---------|------------|
| 논 / 답 | 79 (고정값) |
| 기타 | trunc(23 × AMC2 / (10 + 0.13 × AMC2)) |

소유역 전체의 AMC3는 토지이용별 AMC3의 면적 가중평균으로 계산됩니다.

```
AMC3_유역 = sum(AMC3_i × Area_i) / sum(Area_i)
```

## 트러블슈팅

### 탭 0: PostGIS 연결 실패

**증상**: 실행 후 로그창에 연결 오류 메시지 출력

**확인 사항**:
- 인터넷 연결 및 Azure 방화벽 접근 가능 여부 확인
- psycopg2 라이브러리 설치 여부 확인
- QGIS Python 콘솔에서 `import psycopg2` 실행하여 설치 확인

---

### 탭 0: 처리 시간이 너무 길다

**원인**: 소유역 면적이 넓거나 네트워크 상태가 좋지 않을 경우

**대처**:
- 로그창에서 진행 상황을 확인하며 기다립니다.
- QGIS를 강제 종료하지 마십시오. 재실행 시 처음부터 다시 작업해야 합니다.

---

### 탭 1: 레이어에서 불러오기가 작동하지 않는다

**원인**: 탭 0이 완료되지 않아 "토양군_토지피복_교차" 레이어가 없는 경우

**대처**: 탭 0을 완전히 실행하고 완료 메시지를 확인한 뒤 탭 1로 이동합니다.

---

### 탭 3: CN값이 NULL인 피처가 있다는 경고가 표시된다

**원인 1**: 탭 1에서 일부 원본 이름에 재분류 이름이 매핑되지 않았습니다.
- 탭 1로 돌아가 비어 있는 매핑 항목을 채우고 저장합니다.

**원인 2**: 탭 2의 CN값 테이블에 해당 토지이용분류 항목이 없습니다.
- 탭 2에서 해당 토지이용분류를 추가하고 CN값을 입력합니다.

**원인 3**: 재분류 이름과 CN값 테이블의 토지이용분류명이 정확히 일치하지 않습니다.
- 대소문자, 띄어쓰기, 특수문자를 포함하여 정확히 일치하는지 확인합니다.

---

### 결과 파일이 생성되지 않는다

**확인 사항**:
- 결과 저장 폴더에 쓰기 권한이 있는지 확인합니다.
- 동일한 이름의 results.xlsx 파일이 열려 있으면 닫고 다시 시도합니다.
- openpyxl 라이브러리 설치 여부를 확인합니다.

---

## 유의사항

- 플러그인은 experimental 버전으로, 대용량 소유역 데이터에서 예기치 않은 오류가 발생할 수 있습니다. 처리 전 반드시 원본 데이터를 백업합니다.
