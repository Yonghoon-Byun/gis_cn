# GIS CN값 계산기 — 테스트 환경 셋팅 가이드

## 1. QGIS 설치

### 1-1. 다운로드
- **QGIS 공식 사이트**: https://qgis.org/download/
- **권장 버전**: QGIS 3.34 LTR (Long Term Release) 이상
- **최소 요구**: QGIS 3.0 이상
- Windows용 **OSGeo4W 네트워크 설치** 또는 **독립 설치 패키지(MSI)** 중 선택

### 1-2. 설치 옵션
- 기본 설치 옵션으로 설치 (특별한 설정 불필요)
- 설치 완료 후 QGIS Desktop 실행하여 정상 동작 확인

## 2. Python 라이브러리 설치

플러그인이 QGIS 내장 Python 외에 추가 라이브러리를 사용합니다.
**QGIS의 Python 환경(OSGeo4W Shell)** 에서 설치해야 합니다.

### 2-1. OSGeo4W Shell 실행
- 시작 메뉴 → **OSGeo4W Shell** 검색 → **관리자 권한으로 실행**

### 2-2. 필수 패키지 설치
```bash
pip install psycopg2-binary pandas openpyxl
```

| 패키지 | 용도 | 최소 버전 |
|--------|------|----------|
| `psycopg2-binary` | PostGIS DB 연결 (토양군/토지피복도 조회) | 2.9+ |
| `pandas` | CN값 테이블(xlsx) 읽기/처리 | 2.0+ |
| `openpyxl` | Excel 파일 읽기/쓰기 (결과 내보내기) | 3.1+ |

### 2-3. 설치 확인
OSGeo4W Shell에서:
```bash
python -c "import psycopg2; import pandas; import openpyxl; print('OK')"
```
`OK`가 출력되면 정상입니다.

> **참고**: 독립 설치 패키지(MSI)로 QGIS를 설치한 경우,
> `C:\Program Files\QGIS 3.xx\bin\python-qgis.bat` 을 이용하거나
> QGIS Python 콘솔(플러그인 → Python 콘솔)에서 `import subprocess; subprocess.check_call(['pip', 'install', 'psycopg2-binary', 'pandas', 'openpyxl'])` 실행

## 3. 네트워크 환경 (DB 접속)

플러그인은 PostGIS DB에 직접 연결하여 토양군/토지피복도 데이터를 조회합니다.

### 3-1. DB 접속 정보
| 항목 | 값 |
|------|-----|
| Host | `geo-spatial-hub.postgres.database.azure.com` |
| Port | `6432` |
| Database | `dde-water` |
| User | `waterviewer` |
| Password | `dohwaviewer` |

### 3-2. 네트워크 요구사항
- **포트 6432 아웃바운드** 허용 필요 (Azure PostgreSQL)
- 사내 방화벽/프록시 환경에서는 해당 포트가 차단될 수 있으므로 IT팀에 개방 요청
- VPN 연결이 필요한 경우 VPN 먼저 연결

### 3-3. DB 접속 테스트
OSGeo4W Shell 또는 QGIS Python 콘솔에서:
```python
import psycopg2
conn = psycopg2.connect(
    host="geo-spatial-hub.postgres.database.azure.com",
    port=6432,
    dbname="dde-water",
    user="waterviewer",
    password="dohwaviewer"
)
cur = conn.cursor()
cur.execute("SELECT COUNT(*) FROM public.soil")
print("토양군 레코드:", cur.fetchone()[0])
cur.execute("SELECT COUNT(*) FROM public.land_cover_yangju")
print("토지피복도 레코드:", cur.fetchone()[0])
conn.close()
print("DB 접속 성공!")
```

## 4. 플러그인 설치

### 4-1. 플러그인 폴더 확인
QGIS 플러그인 경로:
```
%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\
```
Windows 탐색기 주소창에 위 경로를 붙여넣으면 바로 이동됩니다.

### 4-2. 설치 방법 (택 1)

#### 방법 A: 폴더 복사
1. 전달받은 `gis_cn` 폴더를 위 plugins 경로에 복사
2. 최종 구조 확인:
```
plugins/
  └── gis_cn/
      ├── __init__.py
      ├── plugin.py
      ├── dialog.py
      ├── cn.png
      ├── cn_value.xlsx
      ├── land_use_mapping.json
      ├── metadata.txt
      ├── core/
      │   ├── __init__.py
      │   ├── db_manager.py
      │   ├── spatial_ops.py
      │   ├── cn_matcher.py
      │   ├── result_calculator.py
      │   └── land_use_mapper.py
      └── reference/
          └── cn_calculator.ui
```

#### 방법 B: 심볼릭 링크 (개발/공유 드라이브 환경)
관리자 권한 명령 프롬프트에서:
```cmd
mklink /D "%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\gis_cn" "전달받은_폴더_경로\gis_cn"
```

### 4-3. 플러그인 활성화
1. QGIS 실행 (또는 재시작)
2. 메뉴: **플러그인** → **플러그인 관리 및 설치**
3. **설치됨** 탭에서 `GIS CN값 계산기` 체크박스 활성화
4. 툴바에 CN 아이콘이 나타나면 설치 완료

> **"실험적 플러그인"이 안 보이는 경우:**
> 플러그인 관리 → **설정** 탭 → "실험적 플러그인도 보기" 체크

## 5. 테스트 데이터 준비

### 5-1. 필요한 입력 데이터
- **소유역 경계 폴리곤** 파일 (SHP 또는 GPKG)
- 좌표계: **EPSG:5186** (Korea 2000 / Central Belt) 권장
- 폴리곤에 **소유역 이름 속성** 컬럼이 있어야 함

### 5-2. 테스트용 체크리스트
| 항목 | 확인 |
|------|------|
| QGIS 실행 정상 | ☐ |
| pip 패키지 3종 설치 완료 | ☐ |
| DB 접속 테스트 통과 | ☐ |
| 플러그인 툴바 아이콘 표시 | ☐ |
| 소유역 SHP/GPKG 파일 준비 | ☐ |

## 6. 빠른 테스트 절차

1. **QGIS 실행** → 툴바에서 **CN값 계산기** 아이콘 클릭
2. **Tab 0 (레이어 불러오기)**
   - "파일 불러오기" 선택 → 소유역 SHP/GPKG 찾아보기
   - 소유역명 컬럼 선택
   - 분류 수준 선택 (대분류 l1 → 빠름, 세분류 l3 → 느림)
   - **실행** 클릭 → 진행바 완료까지 대기
3. **Tab 1 (토지이용 재분류)** — 필요 시 매핑 편집, 기본값으로도 진행 가능
4. **Tab 2 (CN값 편집)** — 기본 CN값 테이블 확인 (수정 없이 진행 가능)
5. **Tab 3 (CN값 계산)**
   - **CN값 계산 실행** 클릭
   - 결과 확인 후 **결과 내보내기** → `results.xlsx` 저장

## 7. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 플러그인이 목록에 안 보임 | 폴더 구조 오류 | `__init__.py`가 `gis_cn/` 바로 아래에 있는지 확인 |
| `ModuleNotFoundError: psycopg2` | pip 미설치 | OSGeo4W Shell에서 `pip install psycopg2-binary` |
| `ModuleNotFoundError: pandas` | pip 미설치 | OSGeo4W Shell에서 `pip install pandas openpyxl` |
| DB 연결 실패 / timeout | 포트 차단 | 6432 포트 아웃바운드 방화벽 허용 요청 |
| "토양군 데이터 없음" | 소유역 범위 밖 | 양주시 인근 데이터로 테스트 (DB: `land_cover_yangju`) |
| 실행 버튼 눌러도 반응 없음 | QGIS Processing 미초기화 | QGIS 재시작 후 재시도 |
| CN 매칭 실패 다수 발생 | Tab 1 재분류 미실행 | Tab 1에서 토지이용 재분류 매핑 후 저장 → Tab 3 재실행 |
